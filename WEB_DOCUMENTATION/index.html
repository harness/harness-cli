    Web Documentation Guide¶

    Generate beautiful web documentation for your Harness CLI using various tools.

    Option 1: MkDocs with Material Theme (Recommended)¶

    Why MkDocs?¶

    • ✅ Beautiful, modern design
    • ✅ Built-in search
    • ✅ Mobile responsive
    • ✅ Dark/light mode
    • ✅ Easy to deploy
    • ✅ No build step needed

    Installation¶

    # Install MkDocs and Material theme
pip install mkdocs mkdocs-material

# Or use pipx for isolated installation
pipx install mkdocs
pipx inject mkdocs mkdocs-material

    Usage¶

    # Serve locally (with live reload)
mkdocs serve

# Open browser to http://127.0.0.1:8000

# Build static site
mkdocs build

# Output will be in site/ directory

    Deploy to GitHub Pages¶

    # One command deployment
mkdocs gh-deploy

# This builds and pushes to gh-pages branch
# Enable GitHub Pages in repo settings

    Customize¶

    Edit mkdocs.yml to: - Change colors and theme - Add/remove navigation items - Configure search - Add custom CSS/JavaScript

    Option 2: Docusaurus (React-based)¶

    Why Docusaurus?¶

    • ✅ Modern React-based
    • ✅ Versioning support
    • ✅ i18n (internationalization)
    • ✅ Plugin ecosystem
    • ✅ Used by Facebook, Meta

    Installation¶

    # Create Docusaurus site
npx create-docusaurus@latest website classic

cd website

# Install dependencies
npm install

    Setup¶

    # Copy documentation
cp -r ../docs ./docs/

# Edit docusaurus.config.js
# Update title, tagline, URLs, etc.

# Start dev server
npm start

# Build for production
npm run build

# Serve production build
npm run serve

    Deploy¶

    # Deploy to GitHub Pages
GIT_USER=<your-username> npm run deploy

# Or use Vercel/Netlify
# Just connect your repo and deploy

    Option 3: Docsify (No Build Required)¶

    Why Docsify?¶

    • ✅ No build step
    • ✅ Single HTML file
    • ✅ Instant setup
    • ✅ Lightweight

    Installation¶

    # Install globally
npm i docsify-cli -g

# Or use npx (no install needed)
npx docsify-cli init ./docs

    Create index.html¶

    Create docs/index.html:

    <!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>Harness CLI Documentation</title>
  <meta name="description" content="Harness CLI Documentation">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <link rel="stylesheet" href="//cdn.jsdelivr.net/npm/docsify@4/lib/themes/vue.css">
</head>
<body>
  <div id="app"></div>
  <script>
    window.$docsify = {
      name: 'Harness CLI',
      repo: 'harness/harness-cli',
      loadSidebar: true,
      subMaxLevel: 3,
      search: 'auto',
      copyCode: {
        buttonText : 'Copy',
        errorText  : 'Error',
        successText: 'Copied'
      }
    }
  </script>
  <script src="//cdn.jsdelivr.net/npm/docsify@4"></script>
  <script src="//cdn.jsdelivr.net/npm/docsify/lib/plugins/search.min.js"></script>
  <script src="//cdn.jsdelivr.net/npm/docsify-copy-code"></script>
  <script src="//cdn.jsdelivr.net/npm/prismjs@1/components/prism-bash.min.js"></script>
  <script src="//cdn.jsdelivr.net/npm/prismjs@1/components/prism-go.min.js"></script>
</body>
</html>

    Usage¶

    # Serve locally
docsify serve docs

# Or with npx
npx docsify-cli serve docs

# Open http://localhost:3000

    Option 4: VitePress (Vue-based)¶

    Why VitePress?¶

    • ✅ Super fast (Vite-powered)
    • ✅ Vue 3 based
    • ✅ Modern design
    • ✅ Great performance

    Installation¶

    # Create VitePress site
npm init vitepress@latest

# Follow prompts
# Choose "docs" as directory

# Install dependencies
npm install

# Start dev server
npm run docs:dev

# Build
npm run docs:build

    Option 5: GitBook (Hosted Solution)¶

    Why GitBook?¶

    • ✅ Hosted solution (no setup)
    • ✅ Beautiful UI
    • ✅ Collaboration features
    • ✅ Analytics
    • ✅ Custom domain

    Setup¶

    1. Go to https://www.gitbook.com/
    2. Sign up for free
    3. Connect your GitHub repository
    4. GitBook automatically renders your markdown
    5. Customize appearance and settings

    Comparison¶

    Tool Difficulty Build Time Hosting Best For
    MkDocs Easy Fast GitHub Pages Quick setup, beautiful docs
    Docusaurus Medium Medium Any Large projects, versioning
    Docsify Very Easy None Any Simple, no-build solution
    VitePress Medium Very Fast Any Vue developers, performance
    GitBook Very Easy None Hosted Non-technical users

    Recommended Setup: MkDocs¶

    Quick Start¶

    # 1. Install
pip install mkdocs mkdocs-material

# 2. Serve locally
mkdocs serve

# 3. Open browser
# http://127.0.0.1:8000

# 4. Deploy to GitHub Pages
mkdocs gh-deploy

    File Structure¶

    harness-cli/
├── mkdocs.yml           ← Configuration (already created)
├── docs/
│   ├── index.md         ← Home page
│   ├── examples.md      ← Examples
│   ├── markdown/        ← Auto-generated command docs
│   │   ├── hc.md
│   │   └── ...
│   └── ...
└── site/                ← Built site (after mkdocs build)

    Customization¶

    Edit mkdocs.yml:

    # Change colors
theme:
  palette:
    primary: blue
    accent: cyan

# Add custom pages
nav:
  - Home: index.md
  - Your Custom Page: custom.md

# Add plugins
plugins:
  - search
  - tags
  - git-revision-date-localized

    CI/CD Integration¶

    GitHub Actions for MkDocs¶

    Create .github/workflows/docs.yml:

    name: Deploy Documentation

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: 3.x

      - name: Install dependencies
        run: |
          pip install mkdocs mkdocs-material

      - name: Deploy to GitHub Pages
        run: mkdocs gh-deploy --force

    Netlify¶

    1. Connect your GitHub repo to Netlify
    2. Build command: mkdocs build
    3. Publish directory: site
    4. Deploy!

    Vercel¶

    1. Connect your GitHub repo to Vercel
    2. Framework: Other
    3. Build command: mkdocs build
    4. Output directory: site
    5. Deploy!

mkdocs serve

    3. Deploy to GitHub Pages: 

      mkdocs gh-deploy

    4. Enable GitHub Pages in repository settings

    5. Share your documentation URL!
