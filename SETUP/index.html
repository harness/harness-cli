    Documentation Setup & Quick Start¶

    What We've Created¶

    Your Harness CLI now has two types of documentation:

    1. Manual Documentation (Current)¶

    High-quality, hand-crafted documentation with: - ✅ Enhanced examples and use cases - ✅ CI/CD integration guides - ✅ Troubleshooting tips - ✅ Best practices

    Location: docs/*.md files

    2. Automated Documentation (Recommended for Future)¶

    Auto-generated from your Cobra command definitions: - ✅ Always in sync with code - ✅ Zero maintenance overhead - ✅ Multiple output formats (Markdown, Man pages, YAML) - ✅ One command to regenerate everything

    Location: docs/markdown/, docs/man/, docs/yaml/

    Quick Start: Automated Documentation¶

    Step 1: Install Dependencies¶

    # Add the cobra doc package
go get github.com/spf13/cobra/doc@v1.9.1
go mod tidy

    Step 2: Generate Documentation¶

    # Generate all documentation formats
make docs

    This creates: - Markdown → docs/markdown/*.md (for websites) - Man pages → docs/man/*.1 (for Unix systems) - YAML → docs/yaml/*.yaml (for tooling)

    Step 3: View Generated Docs¶

    # View markdown
cat docs/markdown/hc.md

# Install and view man pages
sudo cp docs/man/*.1 /usr/local/share/man/man1/
man hc

    How It Works¶

    The cmd/hc/gendocs.go file: 1. Imports all your CLI commands 2. Uses Cobra's built-in doc generator 3. Outputs documentation in multiple formats 4. Runs via make docs command

    Workflow for Future Changes¶

    When You Add/Modify Commands¶

    # 1. Update your command code
vim cmd/artifact/command/new_command.go

# 2. Regenerate documentation
make docs

# 3. Commit both code and docs
git add cmd/ docs/
git commit -m "Add new command with documentation"

    Enhancing Command Documentation¶

    Improve auto-generated docs by enhancing your command definitions:

    cmd := &cobra.Command{
    Use:   "mycommand <arg>",
    Short: "Brief one-line description",  // Shows in command lists
    Long: `Detailed multi-line description.

This command does X, Y, and Z. It's useful for...`,  // Shows in --help

    Example: `  # Example 1: Basic usage
  hc mycommand value

  # Example 2: With flags
  hc mycommand value --flag option`,  // Shows examples

    Args: cobra.ExactArgs(1),  // Validates arguments
}

// Document your flags clearly
cmd.Flags().StringVar(&name, "name", "", "Name of the resource (required)")
cmd.Flags().BoolVar(&force, "force", false, "Force the operation without confirmation")

    Comparison: Manual vs Automated¶

    Aspect Manual Docs Automated Docs
    Accuracy Can become outdated Always current
    Maintenance High effort Zero effort
    Customization Full control Limited styling
    Examples Rich examples Basic examples
    Best for Guides, tutorials Command reference

    Recommended Approach¶

    Use both!

    1. Automated docs for command reference
    2. Run make docs before each release
    3. Commit generated files to git

    4. Use for API documentation

    5. Manual docs for guides

    6. Keep docs/examples.md for tutorials
    7. Maintain docs/index.md as landing page
    8. Add troubleshooting guides as needed

    CI/CD Integration¶

    GitHub Actions¶

    Create .github/workflows/docs.yml:

    name: Documentation

on:
  push:
    branches: [main, v2]
  pull_request:

jobs:
  check-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Go
        uses: actions/setup-go@v4
        with:
          go-version: '1.24'

      - name: Install dependencies
        run: |
          go get github.com/spf13/cobra/doc@v1.9.1
          go mod download

      - name: Generate documentation
        run: make docs

      - name: Check if docs are up to date
        run: |
          if ! git diff --exit-code docs/; then
            echo "❌ Documentation is out of date!"
            echo "Run 'make docs' and commit the changes"
            exit 1
          fi
          echo "✅ Documentation is up to date"

    Publishing Documentation¶

    Option 1: GitHub Pages¶

    # Use docs/ directory as GitHub Pages source
# Settings → Pages → Source: main branch /docs folder

    Option 2: MkDocs¶

    # Install MkDocs
pip install mkdocs mkdocs-material

# Create configuration
cat > mkdocs.yml <<EOF
site_name: Harness CLI
theme:
  name: material
docs_dir: docs
EOF

# Serve locally
mkdocs serve

# Deploy to GitHub Pages
mkdocs gh-deploy

    Option 3: Docusaurus¶

    # Create Docusaurus site
npx create-docusaurus@latest website classic

# Copy docs
cp -r docs/ website/docs/

# Serve
cd website && npm start

    Troubleshooting¶

    "Missing package" errors¶

    go get github.com/spf13/cobra/doc@v1.9.1
go mod tidy

    Documentation not updating¶

    # Clean and regenerate
rm -rf docs/markdown docs/man docs/yaml
make docs

    Man pages not working¶

    # Check man path
man -w

# Try alternative location
sudo cp docs/man/*.1 /usr/share/man/man1/
sudo mandb

    Next Steps¶

    1. Install dependencies: go get github.com/spf13/cobra/doc@v1.9.1
    2. Generate docs: make docs
    3. Review output: Check docs/markdown/ directory
    4. Commit: Add generated docs to version control
    5. Automate: Set up CI/CD to verify docs are current
