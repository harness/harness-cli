    Enhancing Command Documentation¶

    This guide shows how to improve auto-generated documentation by enhancing your Cobra command definitions.

    Adding Examples to Commands¶

    Add the Example field to your command definition:

    Before (Basic)¶

    cmd := &cobra.Command{
    Use:   "generic <registry_name> <package_file_path>",
    Short: "Push Generic Artifacts",
    Long:  "Push Generic Artifacts to Harness Artifact Registry",
    Args:  cobra.ExactArgs(2),
    RunE: func(cmd *cobra.Command, args []string) error {
        // implementation
    },
}

    After (With Examples)¶

    cmd := &cobra.Command{
    Use:   "generic <registry_name> <package_file_path>",
    Short: "Push Generic Artifacts",
    Long: `Push Generic Artifacts to Harness Artifact Registry.

Generic artifacts can be any file type - binaries, archives, configuration 
files, or any other file you want to store and version.`,

    Example: `  # Push a tarball with basic options
  hc artifact push generic my-registry ./app.tar.gz \
    --name my-app \
    --version 1.0.0 \
    --pkg-url https://pkg.harness.io

  # Push with custom filename and description
  hc artifact push generic my-registry ./build/output.zip \
    --name my-build \
    --version 2.1.0 \
    --filename release.zip \
    --description "Production release" \
    --pkg-url https://pkg.harness.io

  # Push configuration file with custom path
  hc artifact push generic config-registry ./config.yaml \
    --name app-config \
    --version 1.0.0 \
    --path configs/production.yaml \
    --pkg-url https://pkg.harness.io`,

    Args:  cobra.ExactArgs(2),
    RunE: func(cmd *cobra.Command, args []string) error {
        // implementation
    },
}

    Adding Global Flags to Commands¶

    To show global flags in command documentation, you need to mark them as inherited.

    Option 1: Use PersistentFlags on Root Command¶

    In your root command (cmd/hc/main.go), flags added with PersistentFlags() automatically appear in all subcommands:

    // In main.go
rootCmd.PersistentFlags().StringVar(&config.Global.APIBaseURL, "api-url", "",
    "Base URL for the API (overrides saved config)")
rootCmd.PersistentFlags().StringVar(&config.Global.AuthToken, "token", "",
    "Authentication token (overrides saved config)")
// etc...

    These will automatically show in all subcommand documentation under "Options inherited from parent commands".

    Option 2: Document Global Flags in Long Description¶

    Add global flags info to the Long description:

    cmd := &cobra.Command{
    Use:   "list",
    Short: "List all registries",
    Long: `Lists all Harness Artifact Registries in your account, organization, or project scope.

The command displays registry information including identifier, package type, size, 
registry type, description, and URL.

Global Flags:
  All commands support these global flags:
  --account string    Account ID (overrides saved config)
  --api-url string    Base URL for the API
  --token string      Authentication token
  --org string        Organisation ID
  --project string    Project ID
  --format string     Output format: table or json (default: table)
  -v, --verbose       Enable verbose logging`,

    RunE: func(cmd *cobra.Command, args []string) error {
        // implementation
    },
}

    Complete Example: Enhanced Command¶

    Here's a complete example showing all best practices:

    func NewPushGenericCmd(c *cmdutils.Factory) *cobra.Command {
    var packageName, filename, packageVersion, description, path string
    var pkgURL string

    cmd := &cobra.Command{
        Use:   "generic <registry_name> <package_file_path>",
        Short: "Push generic artifacts to registry",

        Long: `Push generic artifacts to Harness Artifact Registry.

Generic artifacts can be any file type - binaries, archives, configuration 
files, or any other file you want to store and version. The CLI uploads the 
file with progress tracking and stores it with the specified name and version.

Arguments:
  registry_name       Name of the registry to push to
  package_file_path   Path to the file to upload

The command validates the file exists, uploads it with progress tracking,
and confirms successful upload with metadata.`,

        Example: `  # Basic upload
  hc artifact push generic my-registry ./app.tar.gz \
    --name my-app \
    --version 1.0.0 \
    --pkg-url https://pkg.harness.io

  # Upload with custom filename
  hc artifact push generic my-registry ./build/output.zip \
    --name my-build \
    --version 2.1.0 \
    --filename release.zip \
    --pkg-url https://pkg.harness.io

  # Upload configuration with custom path
  hc artifact push generic config-registry ./config.yaml \
    --name app-config \
    --version 1.0.0 \
    --path configs/production.yaml \
    --pkg-url https://pkg.harness.io

  # Upload with description
  hc artifact push generic my-registry ./app.tar.gz \
    --name my-app \
    --version 1.0.0 \
    --description "Production release v1.0.0" \
    --pkg-url https://pkg.harness.io`,

        Args: cobra.ExactArgs(2),

        PreRun: func(cmd *cobra.Command, args []string) {
            config.Global.Registry.PkgURL = pkgURL
        },

        RunE: func(cmd *cobra.Command, args []string) error {
            // implementation
            return nil
        },
    }

    // Add flags with clear descriptions
    cmd.Flags().StringVarP(&packageName, "name", "n", "", 
        "Name for the artifact (required)")
    cmd.Flags().StringVar(&filename, "filename", "",
        "Name of the file being uploaded (defaults to actual filename)")
    cmd.Flags().StringVar(&packageVersion, "version", "", 
        "Version for the artifact (defaults to '1.0.0')")
    cmd.Flags().StringVarP(&description, "description", "d", "", 
        "Description of the artifact")
    cmd.Flags().StringVar(&path, "path", "", 
        "File path within the package (defaults to filename)")
    cmd.Flags().StringVar(&pkgURL, "pkg-url", "", 
        "Base URL for the Packages API (required)")

    // Mark required flags
    cmd.MarkFlagRequired("pkg-url")
    cmd.MarkFlagRequired("name")

    return cmd
}

    Improving Flag Descriptions¶

    Before (Basic)¶

    cmd.Flags().StringVar(&name, "name", "", "name")
cmd.Flags().StringVar(&version, "version", "", "version")

    After (Descriptive)¶

    cmd.Flags().StringVarP(&name, "name", "n", "", 
    "Name for the artifact (required)")
cmd.Flags().StringVar(&version, "version", "", 
    "Version for the artifact (defaults to '1.0.0')")
cmd.Flags().StringVar(&description, "description", "", 
    "Description of the artifact (optional)")
cmd.Flags().BoolVar(&force, "force", false, 
    "Force upload without confirmation prompts")

    Adding Annotations¶

    You can add metadata to commands for better organization:

    cmd := &cobra.Command{
    Use:   "push",
    Short: "Push artifacts",
    Annotations: map[string]string{
        "group": "artifact-management",
        "help":  "https://docs.harness.io/cli/artifact-push",
    },
}

    After Making Changes¶

    1. Regenerate documentation: 

      make docs

    2. View updated docs: 

      make docs-serve

    3. Check the changes:

    4. Examples now appear in command documentation
    5. Better descriptions for all flags
    6. More helpful Long descriptions

    Best Practices¶

    1. Use field - Always add examples for complex commands
    2. Long field - Provide detailed explanation with context
    3. Short field - Keep it to one line, under 80 characters
    4. Flag descriptions - Be specific about what each flag does
    5. Mark required flags - Use MarkFlagRequired()
    6. Use shorthand flags - Add -n for --name with StringVarP()
    7. Document defaults - Mention default values in descriptions
    8. Add validation - Return helpful error messages
