    Documentation Generation Guide¶

    This guide explains how to generate and maintain documentation for the Harness CLI.

    Automated Documentation Generation¶

    The Harness CLI uses Cobra's built-in documentation generation to automatically create documentation from command definitions.

    Quick Start¶

    Generate all documentation formats:

    make docs

    This will generate: - Markdown files in docs/markdown/ - For websites and GitHub - Man pages in docs/man/ - For Unix/Linux systems - YAML files in docs/yaml/ - Structured data format

    Manual Generation¶

    You can also run the documentation generator directly:

    go run cmd/hc/gendocs.go

    Documentation Formats¶

    1. Markdown Documentation¶

    Generated in docs/markdown/ directory. Perfect for: - GitHub documentation - Static site generators (Hugo, Jekyll, MkDocs) - README files

    Usage: 

    # View in browser or editor
cat docs/markdown/hc.md

    2. Man Pages¶

    Generated in docs/man/ directory. Standard Unix manual pages.

    Installation: 

    # Install man pages system-wide
sudo cp docs/man/*.1 /usr/local/share/man/man1/

# View man page
man hc
man hc-auth-login

    3. YAML Documentation¶

    Generated in docs/yaml/ directory. Machine-readable format for: - API documentation tools - Custom documentation generators - Integration with other systems

    Keeping Documentation Up-to-Date¶

    When to Regenerate¶

    Run make docs whenever you: 1. Add new commands or subcommands 2. Change command descriptions, flags, or arguments 3. Update help text in any command 4. Modify command structure

    Best Practices¶

    1. Update Command Descriptions: Keep Short and Long descriptions in your Cobra commands clear and helpful 

      cmd := &cobra.Command{
    Use:   "mycommand",
    Short: "Brief one-line description",
    Long:  `Detailed multi-line description with examples`,
}

    2. Document Flags: Add clear descriptions to all flags 

      cmd.Flags().StringVar(&name, "name", "", "Name of the resource (required)")

    3. Add Examples: Include examples in the command's Example field 

      cmd.Example = `  # Example 1
  hc mycommand --flag value

  # Example 2
  hc mycommand --another-flag`

    4. Regenerate Before Releases: Always regenerate docs before creating a release

    CI/CD Integration¶

    GitHub Actions¶

    Add to your .github/workflows/docs.yml:

    name: Generate Documentation

on:
  push:
    branches: [main]
  pull_request:

jobs:
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Go
        uses: actions/setup-go@v4
        with:
          go-version: '1.24'

      - name: Generate documentation
        run: make docs

      - name: Check for changes
        run: |
          git diff --exit-code docs/ || \
          (echo "Documentation is out of date. Run 'make docs'" && exit 1)

    Alternative Documentation Tools¶

    Option 1: Cobra Doc Generator (Current)¶

    Pros: - Built into Cobra - Zero configuration - Multiple output formats - Always in sync with code

    Cons: - Basic styling - Limited customization

    Option 2: MkDocs with mkdocs-material¶

    For a beautiful documentation website:

    # Install
pip install mkdocs mkdocs-material

# Create mkdocs.yml
cat > mkdocs.yml <<EOF
site_name: Harness CLI
theme:
  name: material
  features:
    - navigation.tabs
    - navigation.sections

nav:
  - Home: index.md
  - Commands:
    - hc: markdown/hc.md
    - Auth: markdown/hc_auth.md
    - Registry: markdown/hc_registry.md
    - Artifact: markdown/hc_artifact.md
EOF

# Serve locally
mkdocs serve

# Build static site
mkdocs build

    Option 3: Docusaurus¶

    For a React-based documentation site:

    npx create-docusaurus@latest docs-site classic
# Copy markdown files to docs-site/docs/
# Configure docusaurus.config.js
npm start

    Option 4: GitBook¶

    For a hosted documentation platform: - Import your GitHub repository - GitBook automatically renders markdown files - Provides search, versioning, and collaboration features

    Custom Documentation¶

    The manually created documentation in docs/ provides: - Enhanced formatting and styling - Additional context and tutorials - Usage examples and best practices - Links to external resources

    To maintain both: 1. Use make docs for auto-generated reference docs 2. Keep custom guides in docs/ root 3. Link between them as needed

    Documentation Structure¶

    docs/
 index.md                    # Main documentation index (custom)
 examples.md                 # Usage examples (custom)
 DOCUMENTATION.md            # This file (custom)
 markdown/                   # Auto-generated markdown
    hc.md
    hc_auth.md
    hc_auth_login.md
    ...
 man/                        # Auto-generated man pages
    hc.1
    hc-auth.1
    ...
 yaml/                       # Auto-generated YAML
     hc.yaml
     hc_auth.yaml
     ...

    Troubleshooting¶

    Missing Dependencies¶

    If you see errors about missing packages:

    go get github.com/spf13/cobra/doc
go mod tidy

    Documentation Not Updating¶

    1. Ensure you've saved all code changes
    2. Run make docs again
    3. Check that command definitions are correct

    Man Pages Not Installing¶

    # Check man page directories
man -w

# Try alternative location
sudo cp docs/man/*.1 /usr/share/man/man1/
sudo mandb  # Update man database
