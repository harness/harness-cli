# Fix Docker migration MANIFEST_UNKNOWN skip-image bug

## Problem

Daily Docker migration from JFrog → Harness AR marks ~10 images as **failed** with
`MANIFEST_UNKNOWN`. Root cause traced to `module/ar/migrate/migratable/package.go`
`Package.Migrate()` (DOCKER/HELM branch), which calls:

```go
crane.CopyRepository(srcImage, dstImage, craneOpts...)
```

`crane.CopyRepository` (go-containerregistry v0.20.6, `pkg/crane/copy.go`) copies
**all tags in a single `errgroup`**. The first tag whose `puller.Get` returns
`MANIFEST_UNKNOWN` — which happens when JFrog's Artifactory catalog retains an old
tag folder whose `list.manifest.json` still references a platform-manifest digest
(e.g. `linux/amd64`) that was garbage-collected when a newer `-cgr` image was
pushed — cancels the shared `errgroup` context. Every remaining tag aborts and
`g.Wait()` returns that single error, so the whole image is recorded as one
`StatusFail`. This is exactly the reported "skips all remaining tags and marks the
entire image failed" behaviour.

## Fix

Replace the single all-or-nothing `crane.CopyRepository` call with a **per-tag
copy loop** (`crane.Copy` per tag) in a new `Package.migrateOCI` method:

- List source tags with `crane.ListTags`. If listing fails outright → image-level
  `StatusFail` (genuine failure, nothing to iterate).
- Honour no-clobber: when `!Overwrite`, list destination tags once and skip tags
  already present (Pre step already records these as Skipped — avoid double count;
  mirrors `CopyRepository`'s no-clobber behaviour).
- For each remaining tag, `crane.Copy(src:tag, dst:tag)`:
  - success → per-tag `StatusSuccess`
  - stale/orphaned source manifest (`MANIFEST_UNKNOWN`, `MANIFEST_BLOB_UNKNOWN`,
    `BLOB_UNKNOWN`, `NAME_UNKNOWN`, bare HTTP 404) → per-tag `StatusSkip` with a
    descriptive message, **continue**
  - any other error → per-tag `StatusFail`, **continue**
- Error classification isolated in a pure, unit-tested helper
  `isStaleSourceManifestErr(err)`.

Result: one orphaned tag no longer aborts the rest of the image; good tags migrate
and the summary shows exactly which tags were skipped and why.
